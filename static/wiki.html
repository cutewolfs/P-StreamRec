Overview

P-StreamRec monitors supported streaming sources, detects when a configured model or source is live, records the HLS stream with FFmpeg, and makes the finished recording available in the Recordings page. It is designed to run continuously in Docker or on UmbrelOS.

Supported Sources

The app supports provider integrations and direct HLS sources. Site integrations can expose browsing, live status, account follows, and better stream quality when credentials are configured.

Recording Lifecycle

  1. Source registration: a model username or direct m3u8 URL is added through the app or configured through environment variables.
  2. Monitoring: the background monitor checks source status on a recurring interval and skips sources that are offline.
  3. Live detection: when a source becomes available, the resolver prepares the playable HLS URL and stream metadata.
  4. Recording: FFmpeg downloads the HLS segments into an original TS file under the recordings directory.
  5. Conversion: after the stream ends, the conversion task creates an MP4 copy for easier playback and smaller storage.
  6. Library update: the Recordings page discovers the saved files and exposes playback, thumbnails, and management actions.

Accounts and Access

Provider credentials are optional, but they unlock account-specific features such as follow sync and may improve the quality or reliability of stream URLs. App-level password protection is also optional and is controlled by the PASSWORD environment variable.

Store credentials in Docker, Umbrel, or environment configuration. Avoid placing real credentials directly in tracked project files.

Configuration

Most runtime behavior is configured with environment variables. The most common settings are listed below.

Variable Purpose
OUTPUT_DIR Base directory where recordings, thumbnails, and session data are stored.
PORT Web interface port. The default app port is 8080.
FFMPEG_PATH Path to the FFmpeg binary used for recording and conversion.
AUTO_RECORD_USERS Comma-separated list of usernames or sources to monitor automatically.
PSTREAMREC_PROXY_URL Optional outbound proxy for provider requests and compatible stream fetching.

Storage

Recordings are grouped by source name so that each model has a dedicated folder. Original TS files preserve the captured stream, and MP4 files are generated after conversion for convenient playback. 

/data/
  records/
    username/
      YYYYMMDD_HHMMSS_ID.ts
      YYYYMMDD_HHMMSS_ID.mp4
  sessions/
  thumbnails/

TS files are usually larger because they keep the original transport stream. MP4 files are typically smaller and easier to stream in a browser.

Troubleshooting

A model does not appear online
Confirm the username, provider credentials, network access, and whether the source is currently live.
Recording starts but stops quickly
Check FFmpeg availability, provider rate limits, proxy settings, and the app logs in Settings.
Chaturbate requests fail behind protection
Verify that FlareSolverr is running and that its URL is saved in Settings.
Recordings do not show up
Confirm that OUTPUT_DIR points to the mounted data volume and that the app can read it.
